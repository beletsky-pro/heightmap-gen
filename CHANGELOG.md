## v0.2.0 — 2026-04-30

Большой UI-апдейт.

### Дизайн-система в духе HeroUI
- Полностью переписан `src/app.css` под семантические CSS-токены: цвета (фон/панель/инпут/primary/accent), радиусы, тени, переходы.
- Все контролы (кнопки, инпуты, селекты, чекбоксы, range-слайдеры) перерисованы:
  - Скруглённые края 12px
  - Плавные hover/active/focus-states (cubic-bezier 180ms)
  - Focus-ring `box-shadow: 0 0 0 3px primary-ring` на всех интерактивных элементах
  - Range-слайдер с CSS-gradient track-fill (видно сколько проиграно)
  - Кастомный чекбокс с SVG-галочкой
  - Свой select-arrow вместо системного
- Подключены шрифты Inter (UI) и JetBrains Mono (моно) с Google Fonts.
- HeroUI-style topbar: gradient-лого, заголовок, GitHub-иконка, ThemeToggle.

### Темы: light / dark / system
- Новый `src/lib/theme.ts` — store, persist в localStorage, реактивная подписка на `prefers-color-scheme`.
- Новый `src/ui/ThemeToggle.svelte` — segmented control с тремя иконками (солнце / монитор / луна).
- Применяется через `data-theme` на `<html>`. Тема инициализируется в `main.ts` ДО маунта Svelte для избежания FOUC.
- Все цвета во всех компонентах теперь идут через CSS-переменные — обе темы корректны без правок.

### Layout
- Полная высота экрана: `100dvh` + `min-height: 100vh` (поддержка мобильных динамических viewport).
- `body { overflow: hidden }` — нет двойного скролла, скроллится только нужная панель.
- Two-row grid: 56px topbar + 1fr main (3 колонки 320 / 1fr / 300).
- На ширине <= 1100px колонки сжимаются (280 / 1fr / 260).

### Overlay-bar (плоскость/куб/цилиндр/сфера + tile)
- Теперь явно `z-index: 10`, `pointer-events: auto`, фон через `color-mix(...)` и `backdrop-filter: blur(12px)`.
- Виден всегда, даже на тёмных canvas.
- Стилизован как floating segmented controls с тенью.

### Прочее
- Заголовок и подпись приложения вынесены в topbar (раньше были в левой панели).
- Все `<h2>` секций теперь используют общий `.section-heading` token.
- `MapThumbnail` — теперь полноценная focused button с hover-кольцом primary.

## v0.1.6 — 2026-04-30

### B&W маска — отдельный экспортируемый файл
Вместо модификации главной displacement-карты добавлен **отдельный output** `*_mask_bw.png`. Это бинарная (или с регулируемым мягким краем) маска по порогу яркости heightmap.

Для чего: opacity / cutout, stencil, blend между материалами в Max.

UI:
- Новая секция «B&W маска» в левой панели — порог, мягкость края, инверсия.
- 5-й thumbnail в панели «Карты» — кликабельный.
- Кнопка «B&W mask» в секции скачивания.
- «Все карты» теперь скачивает 6 файлов (добавлена маска).

Архитектура: новый `src/shaders/mask.frag`, `generateMask()` и `MaskSettings` в `DerivedMaps.ts`. Маска не влияет на Normal/AO/Roughness.

Из секции «Карта displacement» убран ползунок «Бинаризация» — его роль теперь выполняет отдельная B&W маска.

## v0.1.5 — 2026-04-29

### Чёрно-белые карты для displacement
Раньше heightmap получался серым (значения сжаты к середине). Добавлена пост-обработка:
- **Контраст карты** (0.5..4) — растягивает динамический диапазон от середины.
- **Гамма** (0.3..3) — сдвигает середину к темному/светлому.
- **Бинаризация B&W** (0..1) — мягкий порог 0.5, при 1.0 = чистая B&W.
- **Инверсия** — выпуклое ↔ вогнутое.
- Кнопки **Сброс** и **B&W** (готовый пресет: contrast=2.5 + binarize=0.6).

Все параметры применяются к heightmap **до** деривации Normal/AO/Roughness, поэтому все 5 экспортируемых карт согласованы.

### B&W предпросмотр
Чекбокс «Чёрно-белый предпросмотр» в правой панели — показывает чисто-белую матовую поверхность с displacement, без цвета/AO/нормали. Видно только сам рельеф.

Архитектура: новый `src/shaders/post.frag` + `PostSettings` интерфейс в `TextureGenerator.ts`. Пост-пасс пропускается, если все параметры дефолтные (zero overhead).

## v0.1.4 — 2026-04-29

### Bug fix: экспорт PNG падал
`UPNG.encodeLL is not a function` — в `upng-js@2.1.0` функции `encodeLL` нет (моя ошибка, ориентировался на устаревшую документацию). Текущий релиз UPNG поддерживает только 8-bit RGBA при энкоде, не 16-bit greyscale.

Переключил энкодинг на **fast-png 8.0** — нативно поддерживает 16-bit greyscale, endianness обрабатывается библиотекой.

Бандл уменьшился с 619 КБ до 610 КБ, gzip с 169 КБ до 157 КБ.

## v0.1.3 — 2026-04-29

### Бетон — рандомизация пор и каверн
`poresLayer` переписан: формы теперь сильно вариабельные.
- Анизотропное сжатие со случайным углом — каверны не круглые.
- Радиус варьируется в диапазоне 0.3..1.4 от базового (было 0.6..1.0).
- Изломанный край через две синусоиды разных частот по углу-r.
- Сосед-радиус в цикле увеличен с 1 до 2 — крупные jittered поры не обрезаются.
- Каверны дополнительно деформируются fbm-шумом и получают зернистую внутреннюю структуру.

### Геометрические паттерны — новый класс материалов
Добавлены `Линии`, `Волны`, `Рёбра`. Общие параметры: частота, угол (0/45/90 для бесшовности), скругление края, выпуклость, глубина.
- **Линии** — параллельные приподнятые полосы (ширина линии регулируется отдельно). Пресеты: Опалубка / Тонкие рейки / Гофре / Диагональ.
- **Волны** — синусоидальные с регулируемой резкостью; опц. перекрёстные волны под 90° (рябь-сетка). Пресеты: Мягкая рябь / Стиральная доска / Сетка ряби / Шёлк.
- **Рёбра** — треугольный↔полукруглый профиль, регулируемая ширина ребра. Пресеты: Полу-цилиндры / Острые / Тонкие / Диагональ.

### Удалено
- Материал «Штукатурка» — давал нереалистичный результат.

### 3D-превью
- Растяжение текстуры на цилиндре исправлено через per-shape UV-аспект: цилиндр получает `repeat.x = 2π·r/h ≈ 3.1`, сфера — `repeat.x = 2`. Теперь тайл квадратный на любой форме.
- Свотчи готовых цветов убраны — оставлен только color picker + hex.

## v0.1.2 — 2026-04-29

### Бетон полностью переписан
Старый шейдер давал «общий шум», что не похоже на бетон. Новый имитирует архитектурный/декоративный бетон:
- **Базовая шагрень** — низкочастотный fbm, общая лёгкая неровность.
- **Агрегат-зерно** — высокочастотный fbm, мелкая каменистая шероховатость.
- **Поры (мелкие)** — раковинки 1-3мм от воздушных пузырей; круглые вмятины с мягким краем, плотно распределены.
- **Каверны (крупные)** — большие неровные углубления, форма деформирована fbm-шумом.
- **Волосяные трещины** — две сетки разного размера, смещённые fbm-шумом для извилистости.

Новые ползунки: `Поры`, `Каверны`, `Волосяные трещины`, `Зернистость`. Удалены старые `Шероховатость` и `Трещины` (они не давали правдоподобного результата).

5 пресетов: Гладкий лофт / Архитектурный / Опалубка / Старый / Полированный.

В `lib.glsl` добавлены утилиты `poresLayer` и `hairlineCracks` — переиспользуются для будущих материалов.

### 3D-превью
- Добавлена форма **Цилиндр** (CylinderGeometry, 192 сегмента).
- **Color picker** для базового цвета материала + 4 быстрых свотча (светлый/серый/тёмный/тёплый бетон).
- Регулятор **Глянцевости** (`material.roughness`, шкалирует roughness map).
- Регулятор **Металл** (`material.metalness`).

## v0.1.1 — 2026-04-29

### Bug fixes
- `effect_update_depth_exceeded` при загрузке: `$effect` бесконечно перезапускался, потому что `regenerate()` писал в `$state`-переменные (heightTarget и др.), которые читал же сам через `disposeTargets()`. Обёрнут в `untrack(() => regenerate())`.
- Параметры материала теперь читаются явно через цикл по `currentDef.params` — `void currentValues` не подписывается на nested-мутации в Svelte 5 $state-прокси.

### UX
- Добавлен SVG-favicon (убирает 404 в консоли).

## v0.1.0 — 2026-04-29

Первый релиз. MVP онлайн-генератора heightmap/displacement текстур для 3ds Max.

### Материалы
- Бетон — FBM Perlin + микрозерно + опц. трещины (Voronoi-edges)
- Шпаклёвка — мягкие низкочастотные волны + микрозернистость
- Штукатурка — Voronoi cellular threshold (синхронен с MAXScript Stucco)
- Камень — крупный Voronoi (плиты) + FBM-детали внутри плит

У каждого 6 ползунков и 4 пресета.

### Карты
- Heightmap PNG 16-bit (UPNG.js + big-endian Uint16)
- Heightmap PNG 8-bit
- Normal map (Sobel, тогл OpenGL/DirectX)
- AO (cavity)
- Roughness (база + локальная дисперсия)

Все tileable через 4D-перлин на торе и Voronoi на целочисленной grid.

### UI
- 3D-превью на Three.js (plane / box / sphere, OrbitControls)
- Тогл бесшовного замощения 1×1 / 2×2 / 4×4
- 2D-превью каждой карты с кликом «скачать»
- Размеры экспорта 1024 / 2048 / 4096
- Random seed
- Инструкция «как подсунуть в 3ds Max»

### Стэк
TypeScript + Vite 5 + Svelte 5 + Three.js 0.169 + UPNG.js + vite-plugin-glsl. Деплой на GitHub Pages через `actions/deploy-pages@v4`.
